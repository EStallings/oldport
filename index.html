Projects School & Personal.

Games:

  • Polyomino

    This was a summer project with my friends Joel and Luke. We wanted to make a game, accidentally discovered that moving little blocks around was compelling, and made a game about combining blocks to make larger blocks - well, polyominos. It's fun, kind of addicting, and simple. I did application-level development, the save/load and achievements system, parts of the game logic, and contributed roughly the same to the design process as everyone else.




  • Automaton

    Created as a team project over the course of a semester for a Software Engineering class at UNM, this was originally envisioned by Luke as a SpaceChem-like game to both serve as a brainteaser and potentially help people learn programming. Lots of lessons were learned about doing large applications in Javascript. The end product is confusing to learn, but has solid functionality and can be a lot of fun. And it looks great!

    I wrote the UI library that we used, some of the game logic, and most of the application-level code, as well as the parts that interface with the server backend, and some creative input on design and game mechanics.




  • Siphon

    Created for a two-semester capstone course with IFDM - Interdisciplinary Film & Digital Media, my minor - with a large (~15-20) team, this game was made in the Unity game engine. It's a side-scrolling atmospheric puzzle platformer, in which you play as an alien trying to defend her jungle homeworld from machines trying to steal their water. I formed the initial team and contributed greatly to the initial creative process, though most of this work didn't survive continual revisions later in the development. I did game logic, UI, and sound programming.
